Ruby on Rails CRUD

CRUD stands for Create, Read, Update and Delete data in a database. Active Record automatically allows an application to read and manipulate data stored within tables.

In this tutorial, we will create a Rails CRUD with MySQL database.

Step 1 Create a new Rails application.

  1. rails new crud  

Step 2 Change your directory to crud.

  1. cd crud  

Step 3 Go to the Gemfile in your application and add the following.

  1. gem 'grape'   

Step 4 Go to the config/application.rb file in your application and add the following.

  1. config.paths.add File.join('app', 'api'), glob: File.join('**', '*.rb')  
  2. config.autoload_paths += Dir[Rails.root.join('app', 'api', '*')]  

Step 5 Run the following command:

  1. bundle install  

Step 6 Go to app/views/layouts/application.html.erb and insert the following lines in head tag.

  1. <%= stylesheet_link_tag??? 'https://maxcdn.bootstrapcdn.com/bootstrap/3.3.6/css/bootstrap.min.css' %>  
  2. <%= stylesheet_link_tag??? 'https://cdn.datatables.net/s/dt/dt-1.10.10,r-2.0.0/datatables.min.css' %>  

Step 7 Go to app/views/layouts/application.html.erb and insert the following lines before </body> tag.

  1. <%= javascript_include_tag 'https://maxcdn.bootstrapcdn.com/bootstrap/3.3.6/js/bootstrap.min.js' %>  
  2. <%= javascript_include_tag 'https://cdn.datatables.net/s/dt/dt-1.10.10,r-2.0.0/datatables.min.js' %>  

Step 8 In the above file, replace <%= yield %> with following code:

  1. <div class="container">

  4. <% end %> <%= yield %></div>  
  5. </div>  
  6. </div>

Step 10 Create a controller from the console.

  1. rails g controller products index show new create edit update destroy  

Step 11 Create a model from the console.

  1. rails g model product name:string price:decimal short_description:text full_description:text  

Step 12 Go to app/controllers/products_controller.rb and write the following code.

  1. class ProductsController < ApplicationController   
  2.    
  3.   # GET method to get all products from database   
  4.   def index   
  5.     @products = Product.all   
  6.   end   
  7.    
  8.   # GET method to get a product by id   
  9.   def show   
  10.     @product = Product.find(params[:id])   
  11.   end   
  12.    
  13.   # GET method for the new product form   
  14.   def new   
  15.     @product = Product.new   
  16.   end   
  17.    
  18.   # POST method for processing form data   
  19.   def create   
  20.     @product = Product.new(product_params)   
  21.     if @product.save   
  22.       flash[:notice] = 'Product added!'   
  23.       redirect_to root_path   
  24.     else   
  25.       flash[:error] = 'Failed to edit product!'   
  26.       render :new   
  27.     end   
  28.   end   
  29.    
  30.    # GET method for editing a product based on id   
  31.   def edit   
  32.     @product = Product.find(params[:id])   
  33.   end   
  34.    
  35.   # PUT method for updating in database a product based on id   
  36.   def update   
  37.     @product = Product.find(params[:id])   
  38.     if @product.update_attributes(product_params)   
  39.       flash[:notice] = 'Product updated!'   
  40.       redirect_to root_path   
  41.     else   
  42.       flash[:error] = 'Failed to edit product!'   
  43.       render :edit   
  44.     end   
  45.   end   
  46.    
  47.   # DELETE method for deleting a product from database based on id   
  48.   def destroy   
  49.     @product = Product.find(params[:id])   
  50.     if @product.delete   
  51.       flash[:notice] = 'Product deleted!'   
  52.       redirect_to root_path   
  53.     else   
  54.       flash[:error] = 'Failed to delete this product!'   
  55.       render :destroy   
  56.     end   
  57.   end   
  58.    
  59.   # we used strong parameters for the validation of params   
  60.   def product_params   
  61.     params.require(:product).permit(:name, :price, :old_price, :short_description, :full_description)   
  62.   end   
  63.    
  64. end
